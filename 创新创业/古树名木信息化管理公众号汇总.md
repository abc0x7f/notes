## 软硬件架构
[智慧园林绿化管理信息平台](https://mp.weixin.qq.com/s?__biz=MzkwNTM4ODU3Nw==&mid=2247490590&idx=1&sn=b9dbb281278660b909c2837e7ac72214&chksm=c129d7826ca0efde64b2d01a4caebac8579bc512d585faf2eb154364dad304353ef8e478facc&mpshare=1&scene=23&srcid=04116KCDC0pP2NIHdAlS9Iyi&sharer_shareinfo=ae50584460c89d1799359d320b60f7e3&sharer_shareinfo_first=a7c06a08f111eeb581ae9270df2f8ac2#rd)
平台包括软硬件支撑层、数据层、服务层、应用层以及用户层。
![](1.webp)
1. 软硬件支撑层是平台运行重要支撑环境，主要包括感知设备、传输网络、服务器以及常用的基础软件。感知设备包括视频监控设备、病虫害监测设备及气象监测设备等，主要用于实时数据的收集工作，是智慧园林绿化物联网网格重要的一环，如收集病虫害实时数量、实时的天气温度和湿度情况、人员动态巡查的实时路径等；传输网络包括政务网、物联网、互联网及移动网等，主要用于感知数据的汇交和传输等。
2. 数据层是平台重要的数据支撑，通过构建智慧园林绿化大数据库，在基础地理信息数据的基础上，叠加各类园林绿化专题数据、动态感知数据及多媒体数据等，为平台的应用和领导决策提供翔实、准确的数据服务。

3. 服务层将常用的功能模块进行封装，以服务的方式提供给应用层使用，主要包括地图服务、数据服务、查询统计服务、巡查服务、实时监控服务及视频服务等。
4. 应用层是平台的展示窗口，用户通过应用层提供的功能模块进行可视化管理，主要的功能有园林绿化资源“一张图”管理、电子政务管理、日常养护巡查管理、数字公园管理及辅助决策分析管理等。
5. 用户层是平台应用人员，包括管理人员、设计人员、建设人员以及公众等。
## 数据库建设内容
智慧园林绿化大数据库建设主要包括基础地理信息数据库、园林绿化专题数据库、动态感知数据库、园林绿化业务数据和多媒体数据库。其中基础地理信息数据包括地形图数据、卫星遥感影像数据、园林三维模型数据以及园林全景数据；园林绿化专题数据包括绿地数据、行道树数据、古树名木数据以及公园数据等；动态感知数据库包括日常养护巡查数据、病虫害数据、监测站数据以及气象站数据等；园林绿化业务数据库包括办公数据、审批数据及绿地工程数据等；多媒体数据库包括视频监控数据、图片数据及文本数据等。

## 设计思想
智慧园林绿化管理平台采用SOA设计思想，基于B/S架构，利用JAVA作为开发语言；GIS地图调用和功能开发，采用ArcGIS 10.3及ARCGIS API for JavaScript。平台主要包括的核心功能有园林绿化资源“一张图”管理、电子政务管理、日常养护巡查管理、园林绿化工程管理及辅助决策分析管理。
1. **园林绿化资源“一张图”管理**(园林养护信息图层,园林环境监测信息图层,古树名木图层,网格化管理图层)
2. **园林绿化电子政务管理**个人事务管理(待办工作、完成工作)、日常事务管理、公文管理(收发)及档案管理,业务审批
3. **园林绿化日常养护巡查管理**
4. **园林绿化工程管理**,养护计划管理:养护内容、养护措施、养护时间、养护单位等。养护实施管理:施肥次数、浇灌次数、修剪次数、审核人、审核时间及审核意见.养护巡查管理:巡查类别、问题描述、巡查日期、是否解决以及责任单位等。养护单位信息管理:企业名称、地址、资质以及证书,历史养护记录.
5. **园林绿化工程管理**
6. **园林绿化辅助决策分析管理**:**GIS**空间叠加分析功能和多要素辅助决策功能
